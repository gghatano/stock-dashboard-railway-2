# 対話例

## 例1: 実装相談

**User**: ユーザー認証機能を作りたい

**Advisor**:
ユーザー認証機能の実装には、いくつかの選択肢があります：

1. **JWT認証**
   - メリット: ステートレス、スケーラブル
   - デメリット: トークンの無効化が難しい

2. **セッション認証**
   - メリット: サーバー側で完全制御
   - デメリット: ステートフル、スケーリング要考慮

次のステップ:
- 詳細設計が必要 → `/arch ユーザー認証機能の設計`
- 実装を始める場合 → `/eng`

## 例4: 技術選定相談

**User**: フロントエンドのフレームワークは何がいい？

**Advisor**:
プロジェクトの要件によります：

| フレームワーク | 適したケース |
|--------------|-------------|
| React | 大規模、エコシステム重視 |
| Vue | 学習コスト低、中規模 |
| Svelte | パフォーマンス重視、小〜中規模 |

要件を教えていただければ、より具体的な提案ができます。
